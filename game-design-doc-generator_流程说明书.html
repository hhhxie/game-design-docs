流程说明书

版本：v1.0 · 更新日期：2026-05-11 · 对应 SKILL.md v1.0

一、技能概述

1.1 用途

本技能用于生成专业游戏策划大纲（GDD），遵循行业标准结构（Youdoo 平台规范），覆盖所有游戏类型（动作/休闲/RPG/体育竞技等）。

1.2 触发方式

指令触发：@skill:game-design-doc-generator

关键词触发：

  • 写策划大纲
  • 生成游戏设计文档
  • 帮我写游戏文档
  • game design document
  • GDD

1.3 核心能力

  • 支持两种工作模式：信息预收集 + 细分任务问答
  • 所有文档遵循 references/standard-template.md 的权威 10 节结构
  • 根据游戏类型自动选择参考模板（动作/休闲/体育竞技）
  • 每个问题附带 1-3 个生成选项，降低用户输入负担
  • 自动结构合规检查（商业化/多语言/资产清单/存档逻辑）

二、完整流程概览

【入口判断】用户触发技能 → Phase 0 信息预收集（是否有已有信息/文档？）
【Phase 1】模板选择
加载 references/standard-template.md（必须）→ 加载类型参考模板（按需）
【Phase 2】任务列表创建
根据游戏类型创建细分任务列表（49 项，按类别分组）
【Phase 3】逐任务问答
每个任务问一个问题 → 提供选项 → 用户选择/自定义 → 记录 → 下一任务
【Phase 4】文档生成
编译所有答案 → 结构合规检查 → 写入文件 → 呈现用户
【Phase 5】迭代优化
用户审阅 → 针对性修改 → 更新版本历史 → 最终定稿

三、详细流程说明

Phase 0：信息预收集（可选）

目的：判断用户是否有已有信息，优先使用用户资料生成更精准的选项。

执行动作：

  1. 询问用户是否可以提供已有信息（文档/草稿/参考案例）
  2. 情况 A：用户选择"有" → 读取并分析信息，提取关键要素（游戏类型/核心玩法/美术风格等），后续所有任务的选项优先基于用户提供的信息
  3. 情况 B：用户选择"无" → 直接进入 Phase 1，使用模板作为选项生成基础
优先级来源说明
🥇 1用户提供的信息最高优先级，直接反映用户意图
🥈 2参考文档用户指定的参考资料
🥉 3模板文档作为补充和兜底

Phase 1：模板选择

目的：确定文档结构和内容参考。

执行规则：

  1. 必须加载 references/standard-template.md — 权威文档结构规范，所有游戏类型必须遵循
  2. 按需加载 类型参考模板：
    • 动作/街机类 → references/action-game-template.md
    • 休闲/养成类 → references/casual-game-template.md
    • 体育竞技/足球/多人对战 → references/sports-game-template.md（⚠️ 体育类有专门注意事项）
    • 其他类型 → 标准模板 + 相关章节混合

Phase 2：任务列表创建

目的：将文档生成拆解为可管理的细分任务。

任务分组结构（以动作为例，共 49 项）：

分组任务数内容
基本信息4版本号、作者、游戏ID、更新日期
产品定位6游戏名称、用户年龄、游戏人数、维度、氛围、后期运营
核心玩法4-6玩法概述、主要操控、游戏规则、游戏模式
操作规范3-5基本操作、操作判定、进阶技巧
角色系统4-6角色数量、造型、属性、AI行为
关卡场景4-6场景类型、尺寸、障碍设计、难度递增
道具奖励2-4道具类型、奖励机制
UI界面3-4主菜单、游戏HUD、暂停菜单、结算界面
音效音乐3-4音乐风格、音效设计、语音解说、音频设置
游戏流程3-4开始流程、进行流程、结束流程
美术表现3-4整体风格、角色设计、场景设计、特效设计
后期运营1-2更新计划（可选）

Phase 3：逐任务问答

核心原则：每次只问一个问题，提供 1-3 个生成选项。

📋 **任务 X/49: [任务名称]**
[具体问题]？

**参考选项：**
A. [基于模板/参考/用户信息的选项1]
B. [选项2 — 变体/改进版本]
C. [选项3 — 可选，提供更多可能性]

💡 你也可以直接输入自定义答案

选项生成规则：

  • 选项 A：标准/推荐选择（来自模板）
  • 选项 B：替代方案
  • 选项 C：额外可能性（可选，最多 3 个）
  • 用户可选择选项（A/B/C）或输入自定义答案
  • 用户可说"按模板来"使用默认值

流转规则：

  • 用户回复 → 记录答案 → 标记任务完成 → 进入下一任务
  • 用户一次回复多个答案 → 同时标记多个任务完成
  • 用户说"跳过" → 标记为"待定"，继续下一任务
  • 用户回答模糊 → 追问一个细节，不推进流程

Phase 4：文档生成

编译所有答案 → 按 10 节结构组织内容。

文档必须包含的 10 个一级章节（来自 standard-template.md）：

1. 版本
2. 产品定位（游戏名称/ID/用户年龄/游戏人数/是否3D/游戏氛围/后期运营）
3. 玩法与体感操作（玩法简介/操作规范/基本操作/进阶技巧）
4. 玩法详情（游戏模式设计/属性及计算/角色设计/技能设计/战斗设计/道具与背包/怪物设计）
5. 产品内容与商业化设计（购机附赠/订阅制/买断制）
6. 辅助功能与交互体验（应用导航/准备流程/UX/辅助功能/通用功能）
7. 美术需求（艺术风格/视角/场景/角色/UI/资源清单表格）
8. 音频需求（音乐/音效/语音/资源清单表格）
9. 多语言支持（简中/繁中(台)/英文）
10. 里程碑
备忘录

结构合规检查（必须执行）：

#检查项说明
1章节编号所有主章节必须使用 1. ~ 10. 编号
210 节结构完整缺少第 4/8/9/10 节
3商业化设计必须包含三种商业化模型，不允许为空
4多语言支持必须包含至少 3 种语言
5资产清单表格美术资源清单 + 音频资源清单必须使用表格格式
6存档逻辑每个游戏模式必须说明存档策略

Phase 5：迭代优化

用户审阅 → 提出修改意见 → 针对性修改对应章节 → 更新版本历史。

四、文档质量检查清单

A. 结构完整性（必查项）

#检查项常见遗漏
1章节编号生成版无编号
210 节结构完整缺少第 4/8/9/10 节
3商业化设计完全遗漏
4多语言支持完全遗漏
5里程碑 + 备忘录完全遗漏
6数据与保存无存档逻辑

B. 内容精确度（必查项）

#检查项常见问题
7游戏视角精确描述写"3D视角"而不写"第三人称俯视"
8体感操作判定精度缺失动作编号和判定阈值
9核心机制集中组织带球/换人等分散在多处
10美术资源需求清单缺表格格式
11音频资源需求清单缺表格格式

C. 平台规范合规（必查项）

#检查项说明
13辅助功能 5 个子节应用导航/准备流程/UX/辅助功能/通用功能 均需覆盖
14卡路里计算Youdoo 平台通用功能，必须提及
15产品定位字段完整必须包含全部 7 个字段

五、异常处理规则

情况处理方式
用户回答模糊追问一个细节，不推进流程
用户一次回复多个任务答案同时标记多个任务完成，继续下一未完成任务
商业化章节为空必须补充，默认使用"买断制"
多语言章节为空必须补充，默认"简中/繁中(台)/英文"
缺少资产清单表格必须格式化为表格

六、输出示例

你的游戏叫什么名字？

**参考选项：**

B. 世界杯足球
C. 射门达人

八、注意事项

  • 所有文档必须遵循 standard-template.md 结构，这是权威规范，不可偏离
  • 每个问题只问一次，提供 1-3 个选项，不开放式追问
  • Phase 0 信息预收集 可跳过（用户选 B），但一旦用户选 A，必须优先使用用户提供的信息生成选项
  • 商业化章节绝对不可为空，即使只有"买断制"也要占位
  • 体育竞技类游戏 必须使用 sports-game-template.md，且注意体育类常见错误（视角/球队设计/操作编码/存档逻辑）
  • 文档保存格式：《游戏名称》策划大纲.md
