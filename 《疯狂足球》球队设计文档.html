一、设计概述
🎯

真实性

球队属性参考真实世界杯表现，强队属性高、弱队属性低

⚖️

平衡性

所有球队总属性值（进攻+防守+速度）控制在 195~260 区间，避免碾压式差距

🔀

差异化

每支球队有明确的战术特征（进攻型/防守型/速度型/均衡型），通过阵型和属性侧重体现

🎨

视觉辨识

队服配色严格参考真实国家队球衣，球员肤色贴合实际

属性范围影响机制
进攻 (ATK) 50~95 踢球力度乘数（power × ATK/80）、前锋/中场 AI 进攻积极性（追球距离+2m）、带球时朝向球门速度
防守 (DEF) 50~95 所有球员防守积极性（回防速度+DEF×0.02 m/s）、后卫追球距离+3m、AI 抢球判定范围扩大（+DEF×0.005m）
速度 (SPD) 50~95 跑动速度上限（baseSpeed × SPD/75，base=6 m/s）、加速时最大速度上限、自动追球速度
属性平衡：ATK+DEF+SPD 总和控制在 195~260 之间。顶级球队约 255，中等球队约 230，弱队约 205。属性差距约 20%，确保弱队仍有竞争力。

5人制球队 = 1 守门员 + 4 场上球员，4 个位置：守门员(GK)、前锋(FW)、中场(MF)、后卫(DF)

二、球员AI行为逻辑

本节基于 football_2p.html 原型代码（Player 类 updateAI 方法，约第1386行）整理，描述4种位置角色的AI行为。

2.1 核心参数

参数值说明
AUTO_SPEED1.0 m/sAI基础移动速度
MAX_SPEED6.0 m/sAI最大速度（预留参数）
KICK_RADIUS0.9 mAI踢球/触球判定距离
KICK_POWER_MIN3踢球最小力度（对应最低抛物线高度）
KICK_POWER_MAX25踢球最大力度（对应最大抛物线高度3m）
SWITCH_COOLDOWN0.8 s人类玩家自动切换控制权冷却时间
SWITCH_DISTANCE_RATIO0.6切换滞后系数（防乒乓切换）
DRIBBLE_BALL_OFFSET0.55 m带球时球在身前的偏移距离

2.2 守门员（GK）AI

状态流转：idle → diving（0.3s）→ diving_land（0.6s）→ diving_getup（0.5s）→ idle；飞扑后5秒冷却
展开守门员AI行为规则
持球状态（hasBall = true）：停止移动，球跟随；平滑转向对方球门（每帧15%）；朝向误差<15°时大脚开球（力度18，确保越过中场线）
球在禁区内：快速移向球门线（z = goalLineZ），横向跟踪 ball.x × 0.8，速度3.5
球在禁区外：缓慢移向禁区边缘，速度2.5，横向弱跟踪 ball.x × 0.7
飞扑触发条件：球距球门线<0.5m 且 球距GK<3.6m 且 diveCooldown≤0 且 无人类玩家带球
朝向规则：球在对方半场时面向球；球在我方半场时面向正前方

2.3 前锋（FW）AI

球在进攻半场：积极追球，速度2.5，目标 = 球位置
球在防守半场：回撤保持站位，速度1.0，targetX = ball.x × 0.3，targetZ = attackDir × 6 + ball.z × 0.15
踢球：distToBall < 0.9m 且 kickCooldown≤0，力度8，准度0.7，朝对方球门

2.4 中场（MF）AI

球距 < 12m：积极追球，速度2.0，目标 = 球位置
球距 ≥ 12m：回到中场站位，速度1.0，targetX = ball.x × 0.3，targetZ = ball.z × 0.2
踢球：同前锋（力度8，准度0.7）

2.5 后卫（DF）AI

球在我方半场：积极抢断+解围，速度2.5，目标 = 球位置，解围力度12，准度0.5
球在对方半场：保持防守站位，速度1.0，targetX = ball.x × 0.2，targetZ = -attackDir × 6 + ball.z × 0.15
解围踢球：在我方半场时朝对方球门大范围解围（random offset ±4）

2.6 通用AI踢球物理

powerRatio = (kickPower - KICK_POWER_MIN) / (KICK_POWER_MAX - KICK_POWER_MIN)
targetH = KICK_MIN_HEIGHT + (KICK_MAX_HEIGHT - KICK_MIN_HEIGHT) × powerRatio
vy = sqrt(2 × GRAVITY × targetH)
角色力度准度说明
前锋/中场80.7追求射正，偏移小
后卫（我方半场）120.5全力解围，大范围分散
守门员14~180.5确保越过中场线
踢球冷却：0.5秒；踢球后所有人类玩家 hasBall 状态清零

2.7 人类玩家控制权自动切换

findClosestToBall(candidates)：用距离平方（避免开方）找到离球最近的球员
switchControl：旧球员降级AI，新球员升级人类控制，箭头闪烁3次（80ms间隔）
updateAutoSwitch：每帧调用，对P1和P2分别检测并执行切换
切换条件：switchTimer ≤ 0 且 最近距离² < 当前距离² × 0.6（滞后系数防乒乓）
候选范围：同队所有 role !== 'goalkeeper' 的球员

五、属性影响机制详解

4.1 进攻属性影响

踢球力度 = DEFAULT_KICK_POWER × (ATK / 80)
前锋AI进攻距离 = baseDistance + (ATK - 75) × 0.1  // 基础6m，每点+0.1m
中场AI追球距离 = 12 + (ATK - 75) × 0.2  // 基础12m，每点+0.2m
带球朝向球门速度 = baseSpeed × (ATK / 80)

4.2 防守属性影响

回防速度 = baseReturnSpeed + DEF × 0.02  // 基础1.0 m/s + DEF×0.02
后卫追球距离 = baseDistance + (DEF - 75) × 0.15  // 基础6m，每点+0.15m
AI抢球范围 = DRIBBLE_STICKY_R + DEF × 0.005  // 基础0.8m + DEF×0.005

4.3 速度属性影响

跑动速度上限 = baseSpeed × (SPD / 75)  // base=6 m/s
自动追球速度 = baseChase × (SPD / 75)  // base=1 m/s
加速最大速度 = baseMax × (SPD / 75)  // base=6 m/s
